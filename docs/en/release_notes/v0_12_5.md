# Qamomile v0.12.5

Qamomile v0.12.5 adds Pauli Correlation Encoding (PCE) through `PCEConverter` and `PCEEncoder`, so quadratic optimization workflows can encode many spin variables as `k`-body Pauli correlators on fewer qubits. QURI Parts sampling can now be seeded for reproducible tutorials and benchmarks. This release also hardens controlled-gate execution, measured `Vector[Bit]` condition address resolution on dynamic-control-flow backends, and `expval` observables attached to individual `Vector` elements.

```
pip install qamomile==0.12.5
```

## New Features

### Pauli Correlation Encoding (PCE)

`PCEConverter` implements Pauli Correlation Encoding for quadratic QUBO/Ising optimization problems supplied as either an `ommx.v1.Instance` or a Qamomile `BinaryModel`; higher-order HUBO terms are rejected because the published PCE surrogate assumes pairwise interactions. PCE maps each spin variable to a distinct `k`-body Pauli correlator on a compressed register, choosing the smallest `n >= k` satisfying `C(n, k) * 3**k >= num_variables` unless you pass a larger `num_qubits` (zero-variable inputs return `k`). The converter exposes the assigned observables with `get_encoded_pauli_list()` and decodes final correlator expectation values with `decode(...)`; `BinaryModel` inputs return a `BinarySampleSet`, while OMMX inputs return an `ommx.v1.SampleSet` evaluated through OMMX's own APIs ([#369](https://github.com/Jij-Inc/Qamomile/pull/369)).

```python
from qamomile.optimization.binary_model import BinaryModel
from qamomile.optimization.pce import PCEConverter

ising = BinaryModel.from_ising(
    linear={0: 1.0, 1: -1.0, 2: 0.5},
    quad={(0, 1): 2.0},
    constant=0.0,
)
converter = PCEConverter(ising, correlator_order=2)
observables = converter.get_encoded_pauli_list()
sampleset = converter.decode([0.8, -0.4, 0.2])
print(converter.num_qubits, len(observables))
print(sampleset.samples, sampleset.energy)
```

```text
2 3
[{0: 1, 1: -1, 2: 1}] [0.5]
```

See [Pauli Correlation Encoding (PCE)](https://github.com/Jij-Inc/Qamomile/blob/v0.12.5/docs/en/algorithm/pce_maxcut.ipynb) for a full MaxCut workflow that solves a 20-vertex instance on three qubits.

### Reproducible QURI Parts sampling

`QuriPartsExecutor(seed=...)` and `QuriPartsTranspiler().executor(seed=...)` now use a seeded Qulacs-based sampling path when a seed is provided. Two executors built with the same seed and sampled on the same circuit return identical shot counts, while omitting the seed keeps the previous non-deterministic path. The seed applies only to Qamomile's default Qulacs-based sampler path; custom samplers continue to be supplied through the existing `sampler=` argument ([#441](https://github.com/Jij-Inc/Qamomile/pull/441)).

```python
from qamomile.quri_parts import QuriPartsTranspiler

transpiler = QuriPartsTranspiler()
executor = transpiler.executor(seed=42)
```

## Bug Fixes

- **Measured `Vector[Bit]` element conditions resolve correctly on dynamic-control-flow backends**. After `s = qmc.measure(register)`, Qiskit now lowers patterns such as `if s[i]:`, `if s[0] & s[1]:`, `while s[i]:`, and conditions over sliced views like `if s[j:j+1][0]:` to the correct classical-bit addresses instead of raising segmentation or emit errors. CUDA-Q also covers the scalar `if s[i]:` path, while QURI Parts remains a static-measurement backend and does not support runtime `if` / `while` control flow ([#443](https://github.com/Jij-Inc/Qamomile/pull/443)).
- **`qmc.expval` now binds observables to the right physical qubits for `Vector` elements**. Calls such as `qmc.expval(q[1], obs)` and `qmc.expval((q[1],), obs)` now resolve through the parent `Vector`, sliced views, composite-gate results, and controlled-gate results instead of falling back to the wrong qubit address ([#450](https://github.com/Jij-Inc/Qamomile/pull/450)).
- **Controlled-gate execution is more consistent across local SDK backends**. QURI Parts and CUDA-Q now cover more `qmc.control(...)` patterns end to end, including controlled user kernels, parameterized composites, powered controls, broadcast targets, and selected control indices. CUDA-Q also covers native-gate, stdlib-composite, and Pauli-evolution controlled cases that remain outside QURI Parts' guarded fallback. The expanded cross-backend tests exercise both sampling and expectation-value paths ([#442](https://github.com/Jij-Inc/Qamomile/pull/442), [#444](https://github.com/Jij-Inc/Qamomile/pull/444), [#447](https://github.com/Jij-Inc/Qamomile/pull/447)).
- **Vector-slice refresh and borrow checks are less brittle**. Same-slice refreshes, adjacent symbolic slice borrows, and slice-assignment refresh paths that can be proven safe now compile, while unsupported symbolic overlap cases remain rejected with explicit limitations documented in the repository ([#444](https://github.com/Jij-Inc/Qamomile/pull/444)).
